# ClaudeCodeTest

First project wired up to GitHub via the standard workflow:
new repo per project, changes shipped through feature branches and pull requests
so progress is never lost and unwanted changes can be reverted.

## Workflow

- `/new-gh-project` — turn the current folder into a new public GitHub repo and push it.
- `/ship` — branch the current changes, push, and open a pull request against `main`.
